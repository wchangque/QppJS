# QppJS 开发路线图

本文档是 QppJS 的长期路线图，也是阶段与任务编号的唯一来源。

## 1. 项目目标

QppJS 的目标是：基于 QuickJS 的设计思想，通过 AI 辅助，从零用 C++ 实现一个以学习和理解为优先的 JavaScript 引擎。

优先级如下：
1. 理解 JavaScript 引擎核心组成与 ECMAScript 关键语义。
2. 做出可逐步演进的最小可运行实现。
3. 在语义理解稳定后，再对照 QuickJS 学习工程化取舍。
4. 最后才考虑性能优化、紧凑表示、复杂 GC 与更完整的兼容性。

## 2. 总体原则

- 每个阶段只解决一个核心主题。
- 每个阶段都要有可运行成果。
- 每个阶段都要有最小测试或验证样例。
- 优先做正确、清晰、易理解的实现，不提前追求 QuickJS 风格优化。
- QuickJS 是重要参考，但不是设计裁决的最高依据。

## 3. 阶段总览

- Phase 0：项目骨架与实验环境
- Phase 1：Lexer + Parser + AST
- Phase 2：AST Interpreter
- Phase 3：对象模型第一版
- Phase 4：函数、调用栈、闭包
- Phase 5：原型链、this、new
- Phase 6：Bytecode VM
- Phase 7：异常与复杂控制流
- Phase 8：基础内建对象
- Phase 9：GC
- Phase 10：模块系统
- Phase 11：Promise / Async
- Phase 12：QuickJS 风格优化

## Phase 0：项目骨架与实验环境

### 阶段目标

建立适合长期学习、持续实验和多轮迭代的工程骨架，而不是一开始就堆功能。

### 任务清单

#### 0.1 建立最小目录结构
- 新建 `src/`、`tests/`、`include/`、`docs/` 的基础布局。
- 明确子目录的初步职责。
- 预留后续 `lexer`、`parser`、`runtime`、`vm` 等目录位置。

#### 0.2 建立最小构建链路
- 选定初版构建方式（建议 CMake）。
- 确保空项目可编译。
- 为后续增量添加模块保留入口。

#### 0.3 建立最小 CLI
- 支持命令行传入源码字符串或文件路径。
- 第一版可以只打印源码、token 或 AST。
- 预留 REPL / 脚本执行入口。

#### 0.4 设计错误处理基础结构
- 明确错误分类：语法错误、运行时错误、内部错误。
- 决定初版错误返回风格。
- 建立统一错误输出格式。

#### 0.5 设计第一版 Value
- 支持 `undefined`、`null`、`bool`、`number`、`string`、`object`。
- 优先选择简单实现，不追求 NaN boxing。
- 明确未来可替换底层表示的边界。

#### 0.6 建立测试基线
- 选择并接入最小测试框架。
- 约定 tests 目录结构。
- 让测试命令可跑通。

#### 0.7 建立调试输出入口
- 预留 `dump_tokens` / `dump_ast` 类调试接口。
- 让早期阶段的正确性更容易人工检查。

### 完成标准

- 空项目可构建。
- CLI 可运行。
- 测试框架可执行。
- 有基础 `Value` 与 `Error` 结构。

### 依赖

无。

### 暂不纳入范围

- 执行 JS 语义
- VM
- GC
- 完整标准库

## Phase 1：Lexer + Parser + AST

### 阶段目标

理解源码如何被拆分成 token，并进一步解析成稳定的语法结构。

### 任务清单

#### 1.1 实现 Tokenizer 基础结构
- 定义 token 类型。
- 支持位置信息记录。
- 明确输入流与输出 token 序列接口。

#### 1.2 支持基础词法元素
- 标识符
- 关键字
- 数字字面量
- 字符串字面量
- 标点与操作符
- 空白、换行、注释

#### 1.3 设计 AST 节点体系
- `Program`
- `Statement`
- `Expression`
- `Literal`
- `Identifier`
- `UnaryExpression`
- `BinaryExpression`
- `VariableDeclaration`
- `AssignmentExpression`
- `BlockStatement`
- `IfStatement`
- `WhileStatement`

#### 1.4 实现表达式解析
- 处理运算符优先级与结合性。
- 选定 Pratt parser 或递归下降策略。

#### 1.5 实现语句解析
- `let`
- expression statement
- block
- `if / else`
- `while`

#### 1.6 实现 AST dump
- 提供文本形式 AST 输出。
- 便于人工检查 parser 结果。

#### 1.7 建立 parser 错误报告
- 基础语法错误
- 位置信息
- 最小可读的错误信息

### 完成标准

- 能将基础 JS 子集稳定解析成 AST。
- 能输出 AST dump。
- 能报告基本语法错误及其位置。

### 依赖

- Phase 0 完成构建、CLI、测试与错误基础设施。

### 暂不纳入范围

- 函数声明与调用的完整语义
- 对象字面量
- 模块语法
