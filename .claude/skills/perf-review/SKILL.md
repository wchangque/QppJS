---
name: perf-review
description: 在任意时刻对 QppJS 代码进行性能分析，支持分析单个文件、单个函数或整个项目。输出按严重程度分级的性能问题清单。调用方式：/perf-review（全项目）、/perf-review path/to/file.cpp（单文件）、/perf-review --fn FuncName path/to/file.cpp（单函数）。
---

## 概览

本 skill 在任意时刻启动 `perf-agent` 对代码进行性能分析。支持三种粒度，适合随时检查某个模块或函数的性能状况。

**不需要任何前置状态**，可在开发的任意阶段调用。

---

## 参数解析

| 调用形式 | 分析范围 | 说明 |
|----------|---------|------|
| `/perf-review` | 全项目 | 扫描 src/ 和 include/ 下所有 .cpp / .h 文件 |
| `/perf-review <file>` | 单文件 | 只分析指定文件，路径相对于项目根目录 |
| `/perf-review --fn <FuncName> <file>` | 单函数 | 只分析指定文件中的指定函数 |

解析后向用户确认一句话："将分析 [范围描述]。"

---

## 执行步骤

### 步骤 1：确定分析范围

**全项目模式：**
用 Glob 列出以下路径下的所有文件：
- `src/**/*.cpp`
- `src/**/*.h`
- `include/**/*.h`

输出文件列表，按模块分组（runtime / vm / frontend / base）。

**单文件模式：**
直接读取指定文件，确认文件存在。如果文件不存在，报错退出。

**单函数模式：**
读取指定文件，用 Grep 定位函数名所在行号范围（从函数签名到对应的闭合括号）。
如果函数名不唯一（重载），列出所有匹配，让用户选择后继续。

---

### 步骤 2：启动 perf-agent

启动 `perf-agent`（subagent_type: `perf-agent`），传入：

**全项目模式：**
```
主题：全项目性能分析
审查模式：代码审查

## 分析范围
全量扫描，文件列表如下：
<按模块分组的文件列表>

## 重点关注
- 热路径：VM 主循环（src/vm/）、属性访问、Value 创建与传递
- 已知高频操作：指令分派、对象属性查找、环境变量查找
```

**单文件模式：**
```
主题：单文件性能分析 — <文件名>
审查模式：代码审查

## 分析范围
单文件：<完整文件路径>

## 文件内容
<文件完整内容>
```

**单函数模式：**
```
主题：单函数性能分析 — <FuncName>
审查模式：代码审查

## 分析范围
文件：<文件路径>
函数：<FuncName>（行 N ~ 行 M）

## 函数代码
<函数代码片段>

## 上下文（函数所在类的数据成员定义，如有）
<相关头文件中的类定义摘要>
```

---

### 步骤 3：写入报告文件

**确定文件序号：**
列出 `docs/perf/` 目录下所有已有文件（格式 `NNN-*.md`）。
- 如果目录不存在或为空，序号从 `001` 开始。
- 否则取现有最大序号 +1，补零到 3 位。

**确定文件名：**
根据分析范围生成简短的 slug：
- 全项目：`all`
- 单文件：取文件名去掉扩展名（如 `value_h`）
- 单函数：`fn-<FuncName>`（如 `fn-Execute`）

文件路径：`docs/perf/<NNN>-<slug>.md`

**写入内容：**
在 perf-agent 报告的 `# 性能审查报告` 标题之后、`审查模式` 行之前，插入两行元数据：

```
处理状态：未处理
审查时间：<YYYY-MM-DD>
```

其余内容原样写入。

**向用户展示：**
报告写入后，向用户输出摘要：

```
报告已保存：docs/perf/<NNN>-<slug>.md
新发现问题：P0 N 条，P1 N 条，P2 N 条
```

如果有 P0 问题，追加：
```
⚠️  发现 P0 问题，建议立即处理。是否需要修复建议？
```

如果只有 P1/P2 问题，追加：
```
P1 建议在当前 Phase 内关注，P2 计划在 Phase 12 统一处理。
```

---

## 注意事项

- 全项目模式在代码量大时，perf-agent 可能需要分批读取文件。如果文件列表超过 30 个，按模块拆分为多轮分析，最后汇总结论。
- 单函数模式如果函数体很短（< 10 行），自动扩展上下文到调用该函数的主要调用方（最多 1 层）。
- 本 skill 只做分析，不修改任何代码。如果用户需要修复建议的具体实现，引导使用 `/implement`。
- `处理状态` 字段由人工更新，可选值：`未处理` / `处理中` / `已处理` / `已忽略（含理由）`。
